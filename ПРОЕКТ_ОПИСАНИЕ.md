# AutoService Platform — Описание на дипломен проект

---

## 1. Идея и цел на проекта

**AutoService Platform** е уеб базирана платформа, която свързва автомобилисти с автосервизи. Основната идея е да се опрости и дигитализира процесът от „имам проблем с колата" до „записах час в подходящ сервиз".

Платформата решава три конкретни проблема:

- Потребителите не знаят как да опишат техническия проблем на сервиза
- Трудно е да се намери сервиз, специализиран точно за нужния вид ремонт
- Записването на час е бавно и обикновено се случва по телефон

Решението включва **AI анализ на симптомите** (чрез Google Gemini), който помага на потребителя да формулира проблема и автоматично предлага подходящи сервизи и категория услуга.

## 3. Архитектура

Проектът е изграден като **монорепо** с две отделни приложения:

```
auto-service-platform/
├── client/          ← React фронтенд (работи на порт 5173)
└── server/          ← Express бекенд (работи на порт 5000)
```

### Комуникация

Фронтендът комуникира с бекенда чрез **REST API** (`/api/...`). В режим на разработка Vite проксира заявките, така че няма CORS проблеми.

### Автентикация

Използва се **JWT (JSON Web Token)** с два токена:

- **Access token** (15 мин.) — изпраща се в `Authorization: Bearer ...` хедъра
- **Refresh token** (7 дни) — съхранява се в httpOnly cookie (недостъпен от JavaScript — защита от XSS)

При изтичане на access token-а, клиентът автоматично прави refresh заявка и повтаря оригиналната заявка.

## 5. База данни — модели

MongoDB се използва с Mongoose. Всеки модел е описан като TypeScript интерфейс.

### User (Потребител)

```
email, password (bcrypt), firstName, lastName,
role (user | service_owner | admin),
avatarUrl, refreshToken, resetPasswordToken, resetPasswordExpires
```

### Vehicle (Автомобил)

```
ownerId → User, brand, model, year, engine,
fuelType (petrol | diesel | electric | hybrid | lpg),
transmission (manual | automatic),
vin, registrationNumber, mileage, imageUrl
```

### ProblemReport (Доклад за проблем)

```
userId → User, vehicleId → Vehicle,
title, description, status (open | matched | booked | closed),
aiSummary, aiUrgency (low | medium | high | critical),
aiSuggestedCategories[], aiQuestions[], aiBriefForShop
```

### ServiceCenter (Сервиз)

```
name, description, address, city, region, phone, email,
servicesOffered[], workingHours { open, close, days[] },
ratingAvg, reviewCount, isApproved, createdBy → User
```

### AvailableSlot (Свободен час)

```
serviceCenterId → ServiceCenter, date, time, isBooked
```

### Booking (Резервация)

```
userId → User, vehicleId → Vehicle,
serviceCenterId → ServiceCenter,
problemReportId → ProblemReport (опционално),
slotId → AvailableSlot,
bookedDate, bookedTime, status (pending | confirmed | cancelled | completed),
note, aiBriefSnapshot
```

### Review (Оценка)

```
userId → User, serviceCenterId → ServiceCenter,
bookingId → Booking, rating (1–5), comment
```

## 7. Сигурност

- **Пароли** — хешират се с bcrypt (12 rounds) преди запис в базата
- **JWT** — access токените са краткотрайни (15 мин.); refresh токените са в httpOnly бисквити
- **Role-based access** — middleware проверява ролята на потребителя за admin операции
- **Валидация** — всички входни данни се валидират с Zod и на сървъра, и на клиента
- **Токен за нулиране на парола** — SHA-256 хеш, валиден 1 час
